# Archiwum dokumentacji

Archiwum zachowuje wybrane badania, audyty i decyzje, których kontekst nadal może być użyteczny. Nie służy jako aktywny backlog ani źródło bieżącego stanu.

Zakończone plany wykonawcze, checklisty jednorazowych sprintów i raporty będące kopią historii commitów są usuwane. Nadal można je odzyskać z Git.

Jeżeli dokument w archiwum przeczy aktualnym plikom, obowiązuje kolejność:

1. `../HANDOFF.md`
2. `../plan-sprintow-2026-07.md`
3. `../roadmap.md`
4. `../wizja-i-plan-produktu-v2.md`

Do archiwum trafiają wyłącznie materiały o trwałej wartości badawczej lub decyzyjnej. Zwykły raport „co wdrożono” powinien zostać zapisany w kodzie, testach, changelogu Git i krótkim bieżącym handoffie.
